블록체인 용어 정리
블록, 블록체인, 분산원장 - p2p
거버넌스, 컨소시엄 - 메인넷, 정책, 투표, 50%
트랜잭션 - 블록체인의 데이터
암호화폐 - 수수료 보상, 마이닝 채굴 => 네트워크 운용을 위한, 전자적 처리를 위한 데이터
합의알고리즘 - 작업증명(POW), 지분증명(POS) - 누가 어떻게 데이터를 처리 하는가 !
네트워크, P2P, 서버-클라이언트
피어(private), 노드(public) - 블록체인 참여자 노드(identity를가짐 => certificate, public, private키를 가짐)
비대칭키, 개인키, 공유키, 해쉬, 머클 트리 - 암호화 복호화 (public키를 이용해 암호화, private키를 이용해 복호화),{account(public키), key(private키)}
	- 머클트리 = 이전의 해쉬값을 포함해 새로운 해쉬값을 만들어 다음 노드에 넘겨준다.
	1. 암호통신
	2. 서명
비잔틴 장군 문제 알고리즘(BFT) - 분산시스템, 동기화에서 자주 쓰임
	A,B,C,D장군 4명이 있는데 그 중 B가 리더이고, A가 B에게 트랜잭션을 보냈다.
	B가 C,D한테 트랜잭션을 나눠주고, C,D가 A에게 검증을 요청하고, 검증이 완료되면 모든 장군이 트랜잭션을 추가한다.
스마트 컨트랙트, 체인코드 - 협업프로세스, 프로그램
플랫폼, SDK, API - eather, fabric, Android OS, 등 실행환경 개발환경, 프레임워크
멀티체인 - 블록체인이 여러개로 이어져있는데 가장긴(깊이) 블록체인이 선택 된다
하드포크, 소프트포크 - 합의로 의해 잠시 중단 하고 다시 재개(소프트포크), 합의 없이 이전버전으로 다시재개(하드포크 => Ether, Ether classic)
WEB3, JSON, JSON-RPC - 블록체인 분산시템을 이용해 APP에 연결시켜주는 인터페이스(WEB3)
			JavaScript로 이루어져 있는 key와 value로 이루어 져 있는 프로그램(Json)
			분산시스템을 이용할 때 작업 요청이나 결과(Remote Process Call)를 보여주는 인터페이스

VM, JVM, EVM, CCENV - 스마트 컨트렉트가 동작할 수 있도록 도와주는 실행환경
ONCHAIN, OFFCHAIN, SIDECHAIN - 한개로 이루어져 있는 mainchain(ONCHAIN)
				다른 DB에 해쉬 값을 저장해 블록체인에 넣어준다.(OFFCHAIN)
				이전 트랜잭션들을 포함시켜준다, TPS(트랜잭션이 처리되는 속도)를 높여준다. 복잡도는 높아지고, 관리도 어려워짐(SIDECHAIN)
